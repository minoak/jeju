# Jeju POP Trail — 프로젝트 현황 & 병렬 작업 분해

> 마지막 업데이트: 7/7. 설계 확정 + 파이프라인 세로 관통(스모크 테스트) 완료 상태.

## 1. 한 줄 정의

유튜브 영상/숏츠를 근거로 제주 여행 질문에 답하는 RAG 서비스.
질문을 던지면 실제 영상 카드를 근거로 **"오늘의 셋리스트(코스)"를 발매**한다.

차별점: 공식 관광 DB·블로그에 없는 롱테일 스팟 + 모든 답변에 출처 영상 링크(환각 방어).

## 2. 확정된 설계 (변경 시 전체 공유 필수)

**파이프라인 (인덱싱, 배치):**
```
수집(유튜브 API, 키워드 격자 41개, 무손실)
→ Pass 1 (gpt-5-mini: 스팟 추출 — name/region/category/tags/summary/address/info_richness)
→ 카카오 로컬 (실존 검증 + 좌표 + place_id + 정식명칭) ← Pass 2를 지도 DB 조회로 대체
→ 병합 (place_id 기준, mention_count 집계, 언급전용 레코드 흡수)
→ 임베딩 (text-embedding-3-large, summary만 / low 등급 제외) → Chroma
```

**런타임 (서빙, 3단계 체인 — "에이전트"가 아니라 구조화된 워크플로):**
```
[1] 질문 분석 (LLM: 무드/밀도 표현 → 검색어 번역, 지역/카테고리 추출)
[2] 벡터 검색 (Chroma top-k + region/category 메타데이터 필터)
[3] 셋리스트 생성 (LLM: 카드만 근거로 코스 구성, 지어내기 금지)
```

**스택:** Streamlit 단일 앱 → Docker → Cloud Run / GitHub Actions CD (main 머지 시 자동 배포)

**컨셉:** K-POP 앨범 발매 문법 (다크 스테이지 + 핑크/시안, 트랙 카드, "차트인 N회" = mention_count).
디자인 기준: `jeju_pop_trail_design.html` 참조.

| 셋리스트 생성 [3] | ✅ 스모크 통과 | 근거 충실성 프롬프트 초안 있음 |

| Hit@5 평가 | ⬜ 미착수 | 골든셋 의존 |
| 레포/CD | 🔶 스캐폴드 완료 | GCP 서비스계정 + Secrets 등록 남음 |

## 4. 병렬 작업 트랙 (의존 관계 명시)

**트랙 A — 데이터 파이프라인** (순차 의존, 1인 전담 권장)
Pass 1 전량 → 카카오 검증/좌표 → 병합 → 임베딩 → chroma_db 산출.
산출물: `data/processed/cards.json` + `chroma_db/`
다른 트랙은 이 완료를 기다리지 않는다 (아래 계약 참조).

**트랙 B — 앱/UI** (트랙 A와 완전 병렬 가능)
Streamlit으로 질문 입력 → 셋리스트 화면. 디자인 초안 HTML의 톤 이식.
**mock 카드 20장(스키마 계약 준수)으로 즉시 시작** — 실데이터는 나중에 파일 교체만.

**트랙 C — 평가** (트랙 A와 병렬, 오히려 먼저)
골든 질문셋 25문항 작성: 직접형/무드형/밀도형/지역조합형 × 6~7개.
⚠ 반드시 카드 데이터를 안 본 사람이 작성 (자기확인 오염 방지).
이후: Hit@5 스크립트, [3]단계 루브릭 채점표 (근거충실성/적합성/구성).

**트랙 D — 인프라** (병렬)
GCP 서비스계정 발급 → GitHub Secrets 등록 → 스켈레톤 앱으로 CD 관통
("빈 앱이라도 URL이 뜬다"를 최우선 확보 — 배포 리스크 앞당기기).

## 5. 인터페이스 계약 — 카드 스키마 (병렬 작업의 고정점)

트랙 B(UI)와 트랙 A(데이터)가 이 스키마만 지키면 서로 기다릴 필요 없음.
**변경은 전원 합의로만.**

```json
{
  "spot_name": "바다다",
  "official_name": "바다다",          // 카카오 정식명칭 (병합 후)
  "place_id": "12345678",             // 카카오 place_id (병합 키, 없으면 null)
  "region": "중문",
  "category": "카페",                  // 단일값: 카페|맛집|포토스팟|자연|기타
  "tags": ["비치클럽", "수영장", "오션뷰"],
  "summary": "수영장과 오션뷰를 갖춘 카페 겸 펍…",
  "address": "제주 서귀포시 …",        // null 가능
  "lat": 33.24, "lng": 126.41,        // null 가능
  "geo_confidence": "high",            // high(주소)|mid(이름검색)|none
  "info_richness": "high",             // high|mid|low (low는 임베딩 제외)
  "mention_count": 4,                  // 차트인 횟수
  "video_ids": ["2UnsSMFYd1M", "..."],
  "view_count_max": 822942
}
```

UI 참고: 썸네일 = `img.youtube.com/vi/{video_id}/mqdefault.jpg` (API 불필요),
영상 링크 = `youtube.com/watch?v={video_id}`.
