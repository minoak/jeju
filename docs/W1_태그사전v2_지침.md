# W1 지침 — 태그 사전 v2 + 번역기 (클로드 코드 작업 계약)

> 배경: `백엔드개편_지시서_v2.md`의 W1. 임베딩의 역할 전환 — "카페 코퍼스 검색"에서
> **"사용자 언어 → 통제 어휘 번역"**으로. 임베딩 공간에 카페 1,064곳이 아니라 태그 수십 개를 넣는다.
> W0(정본 병합)과 병렬 가능 — 단, 아래 잠금 파일 절대 불가침.

## ⚠ 잠금 파일 (다른 세션 작업 중 — 수정 금지)

`pipeline/merge.py` `pipeline/embed.py` `app/server.py` `FRONTEND_지침.md` `HANDOFF.md`
그리고 `pipeline/evidence_polarity.py`, `근거극성_지침.md` (코워크 관리분 — 읽기는 가능).
**이 작업의 산출물은 전부 신규 파일이다.** 기존 파일 수정이 필요해 보이면 멈추고 보고.

- 현행 사전: `pipeline/naver_refine.py`의 `CAFE_TAGS` 20개 (수정하지 말 것 — v2는 별도 파일)
- 태그 분포 실측 (7/9, `네이버 정제.jsonl` 996곳):
  - 사용 빈도: 주차편함 790 / 감성 670 / 오션뷰 387 / 베이커리 336 / 웨이팅 226 / 대형 219 /

  - tags_extra 승격 후보: 소품샵겸업 25(→category와 중복, 승격 제외) / 포토존 22 / 핸드드립 11 /

## 산출물

### 1. `data/processed/태그사전v2.json`
```json
{
  "version": "v2-2026-07-09",
  "tags": [
    {"tag": "노을", "synonyms": ["선셋", "일몰", "노을 맛집", "석양"],
     "hard_capable": false, "exclude_for": null, "status": "기존"},
    {"tag": "애견동반", "synonyms": ["반려견동반", "반려동물 동반", "강아지 동반", "펫 프렌들리"],
     "hard_capable": true, "exclude_for": null, "status": "기존"},
    {"tag": "노키즈존", "synonyms": ["노키즈"],
     "hard_capable": true, "exclude_for": "아이 동반 쿼리", "status": "기존"},
    {"tag": "포토존", "synonyms": ["사진 명소", "인생샷", "포토스팟"],
     "hard_capable": false, "exclude_for": null, "status": "승격(빈도22)"}
  ]
}
```
- 기존 20개 전부 + 승격 확정(포토존/핸드드립/한라산뷰) + 동의어 통합.
- 빈도 5~9 후보(포장/정원/가성비/빈티지)는 `"status": "보류"`로 넣되 임베딩에서 제외 — 민옥 승인 대기.
- synonyms는 태그당 3~6개, "사용자가 실제로 칠 법한 표현"으로 (검색 쿼리 상상력).
- **hard_capable 판정 원칙**: 가부 판정형(있다/없다)만 true — 애견동반, 노키즈존, 키즈친화, 포장.
  ⚠ 주차편함은 가부형이지만 **hard_capable=false로 시작** — 79%(790/996) 부여율이라 변별력·신뢰도
  의심 상태. tag_audit(근거극성 배치의 부산물) 결과로 재평가 예정. 사전에 주석으로 남길 것.
- **exclude_for(배제 의미론)**: 노키즈존은 "아이와 함께" 류 쿼리에서 하드 **배제** 필터.
  하드조건에는 요구만이 아니라 배제도 있다.

### 2. `pipeline/tag_embed.py`
- 태그사전v2의 tag + synonyms 전부를 text-embedding-3-large로 임베딩 (코퍼스와 모델 통일)
- 출력: `data/rag/태그사전_임베딩.npz` (벡터 행렬 + 라벨 배열). 수십 벡터 — Chroma 불필요, npz면 충분
- 키: `.env`의 `OPENAI_KEY` (naver_refine.py의 env 로딩 패턴 재사용)

### 3. `app/translate.py`
```python

    # 2) 임베딩 lookup: cos 유사도 최고 태그, threshold 미달이면 실패
    # 3) 실패 시 {"tag": None, "method": "unresolved"}
    # 반환: {"tag": "노을", "method": "exact|embedding|unresolved", "score": 0.87, "input": term}
```
- threshold는 하드코딩하지 말고 상수 + 실측 근거 주석. 반환 dict가 곧 번역 로그의 한 줄이 된다.
- **강제 번역 금지** — 애매하면 unresolved가 정답 (정직한 실패 원칙).

### 4. 검증 (완료 기준)
- `eval/태그번역_검증.py` (신규): ⓐ 전체 태그×synonym 유사도 행렬을 CSV로 출력
  (`eval/태그유사도행렬.csv`, utf-8-sig) — 사람이 눈으로 이상 매핑 확인용. 공간이 작아 전수 육안 가능.
  ⓑ 테스트 표현 30개("노을 맛집"→노을, "강아지랑"→애견동반, "주차 편한 데"→주차편함,
  "조용한 카페"→조용함, "바다 보이는"→오션뷰, 그리고 unresolved여야 하는 것 5개 포함)
  통과표 출력. 실패 케이스는 고치지 말고 표에 남길 것 — threshold 튜닝 근거.

표본은 random.sample / 엑셀 CSV는 utf-8-sig / 경로는 ROOT 상수 / except는 반드시 출력.
