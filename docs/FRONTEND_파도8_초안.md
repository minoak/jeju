# 프론트 파도 8 초안 — "이렇게 찾았어요" (검색 설명가능성 + 정직한 0건)

> 작성: 코워크(SilenceBreaker) 2026-07-09 밤. ⚠ **초안** — W2~W4 코어 빌드 진행 중, W5(server 통합) 후 필드명 최종 확정.
> `FRONTEND_지침.md`는 잠금 목록이라 별도 파일로 작성 — 잠금 해제 후 파도 8로 편입.
> 계약 원칙 동일: **필드는 추가만(하위호환), 없으면 렌더링하지 않는다, spot_name 조인 키 불변.**

## 배경 (왜 바뀌나)

백엔드 개편 v2: "검색엔진에 LLM을 붙인 게 아니라, LLM에게 검색 도구를 줬다."
- 몸통(검색·정렬·완화)은 전부 코드 — 각 단계가 자기 결정을 기록(TraceState)
- LLM은 양끝 2회만: router(의도 분류) + 종합(근거 인용 추천문, 3규칙 코드 강제)
- **정직한 실패가 컨셉**: 0건이면 0건이라고 말하고, 완화했으면 완화했다고 라벨로 밝힌다

## 8-1. /search 응답 최상위 신규 필드

```jsonc
{
  "query": "...", "region": "...", "browse": false, "total": null,   // 기존 동일
  "cards": [ ... ],                                                   // 기존 동일

  "how": {                                    // ★ 신규 — "이렇게 찾았어요" (없으면 미표시)
    "translation": [                          //   사용자 표현 → 태그 번역 로그
      {"input": "노을 맛집", "tag": "노을", "method": "exact"}
    ],
    "unresolved": ["물멍"],                    //   해석 못 한 조건 (강제 번역 안 함 — 정직)
    "relaxation": [                           //   완화 이력 — label이 그대로 사용자 문구
      {"action": "soft_drop", "condition": "조용함", "label": "조용함 조건을 빼고 찾았어요"}
    ],
    "region_expanded": "한림",                 //   지역 확장했으면 그 버킷 (아니면 null)
    "funnel": [                               //   깔때기 (개발·데모용 — 접힌 UI 권장)
      {"stage": "전체", "n": 831}, {"stage": "region:애월", "n": 87}, {"stage": "tags:노을", "n": 12}
    ]
  },

  "honest_zero": false,                       // ★ 신규 — true면 "조건에 맞는 카페 없음" (에러 아님!)

  "intro": "…",                               // ★ 부활 (파도 7에서 은퇴했던 자리 재점등)
  "notice": "'물멍' 조건은 해석하지 못했어요"   // ★ 신규 — 번역·완화 고지 (코드 강제 삽입분 포함)
}
```

- **`intro`/카드 `reason` 부활**: 파도 7의 폴백 로직(`res.intro || 자체 문구`, `c.reason || firstSentence`)이
  그대로 다시 켜진다 — **프론트 무수정 동작**. 단 이번 LLM 종합은 파도 7 은퇴 전과 다름:
  인용은 카드 원문 부분문자열 검증(결정 26), 인기 수치 입력 금지(결정 25), 실패 시 코드 폴백.
  종합층은 **선택층** — 꺼지면 intro/reason 없이 matched(파도 7)로만 동작. 렌더 규칙 동일: 없으면 안 그린다.
- **지연**: 종합층 켜지면 응답이 1~3초 → 3~6초대로 복귀 예상. 파도 3의 로딩 UI 그대로 유효.

## 8-2. UI 표시 제안 (우선순위순)

1. **완화 라벨 배너** (제일 중요 — 신뢰 컨셉의 화면 번역):
   `how.relaxation`이 있으면 결과 상단에 눈에 띄는 안내 배너 — label 원문 그대로.
   "조용함 조건은 빼고 찾았어요" → 사용자가 "이건 타협된 결과"임을 알고 본다.
   `region_expanded`도 같은 배너 스타일: "협재 밖 한림까지 넓혀서 찾았어요".
2. **정직한 0건 화면**: `honest_zero: true`면 빈 결과를 에러/빈화면이 아니라
   "조건에 맞는 카페를 찾지 못했어요" + (relaxation 있으면) 라벨 붙은 대안 섹션으로.
   **0건이 정상 응답이라는 게 이 개편의 핵심** — 프론트가 이걸 실패처럼 그리면 컨셉이 죽는다.
3. **unresolved 고지**: `how.unresolved`가 있으면 캡션 — "'물멍' 조건은 해석하지 못했어요".
   몰래 무시하지 않는다는 표시. notice 필드에 문장이 오면 그걸 그대로 써도 됨.
4. **"이렇게 찾았어요" 접힘 상자**: `how.translation` 요약 한 줄
   ("'노을 맛집'은 노을 태그로 해석") — 접힌 상태 기본, 펼치면 funnel 깔때기
   (전체 831 → 애월 87 → 노을 12 → 상위 5). **발표 스크린샷 재료이기도 함.**
5. **배지 우선순위**: name_match(파도 1) > closed(파도 7) > 완화 배너(파도 8) > 여론 뱃지(파도 5·6).
   충돌 시 이 순서로.

## 8-3. /trace 엔드포인트 (예정 — W5)

`GET /trace?q=...` = TraceState 전문 (개발·데모 전용, 프론트 필수 아님).
`?debug=1` 관례(파도 7의 유리상자)와 동일 계열 — 데모 페이지에서 깔때기 시각화에 쓸 수 있음.
