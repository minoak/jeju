## 오늘 한 줄

**"검색이 대답하는 법을 배웠다"** — 이름을 치면 그 카페가 나오고(조회), 빈 질의엔 지분으로 답하고(브라우즈),
조건 질의엔 이유를 달아 답하고(LLM 선별), "그래서 누가 뭐라고 했는데?"엔 원문 인용으로 답한다(/evidence).

## ① 해지개 사건 — 이름 매치 레이어

**증상**: 유명 카페 "해지개"를 검색하면 top 10 전멸 (1위 유사도 0.328 = 노이즈). 데이터엔 있는데(정제 high·판정 유지·임베딩 3건) 검색이 못 찾음.
**원인**: ⓐ 임베딩 문서에 카페명이 없음 ⓑ 근본적으로 고유명사 조회는 임베딩의 직업이 아님.
**처방**: 서빙 코퍼스에서 이름 사전 구축(815건) → 질의에 카페명 포함 시 무조건 1순위 고정(`name_match`). 코드 몇십 줄, LLM 0콜.
**후속 사건**: "애월 카페" 검색 → "애월카페"라는 상호가 1위로 (아이러니). 스톱워드만으로 조립된 이름 11건 사전 제외.
  단 이건 하드코딩 방어선 — 근본 해결은 의도 분류(조회/탐색)이고 LLM [1]층이 승계 예정.

**발표 포인트**: 질의 유형이 두 종류인데 도구가 하나였다. 층마다 직업이 다르다 — 조회(사전)/소집(임베딩)/판단(LLM)/검증(코드).

## ② 지역 오염 24.6% — 주소 기반 교정

**실측**: chroma의 region 라벨(Pass 1 LLM 추측)을 네이버 주소와 대조 → 주소 보유 852곳 중 **불일치 210곳(24.6%)** + 한경면은 버킷 자체가 없음(14곳). 오른(성산 해안)이 '제주시내', 레이지펌프(애월)가 '제주시내'.
**처방**: 지역은 주소에서 코드로 유도(원칙 5의 연장 — 지역도 식별자). 2단 계층: 버킷(읍면급)/세부(리·동급), 880곳 적용.
  필터는 chroma where 대신 교정값 코드 필터 + 단계 완화(세부→버킷→전체).
**효과**: "성산 오션뷰"에 카페 오른이 돌아옴. "협재"는 협재리 우선 + 한림읍 채움(계층 필터).

**발표 포인트**: LLM 추측 라벨을 주소가 교정 — "지역도 식별자다, 식별자는 LLM 우회".

## ③ 브라우즈 모드 — 빈 질의에는 지분이 순위다

**문제(민옥 실사용)**: "애월 카페" → 5개만, 순서 근거 없음. 해지개(블로거 79) 미노출에 "이유가 있어야" 지적.
**진단**: 지역+업종만 있는 질의는 의미가 빈 질의 — 유사도 정렬은 노이즈. 모든 카페가 동점이고, 동점 정렬은 원칙 8이 정한 인기 신호의 자리.
**처방**: 스톱워드 걷어내고 알맹이 없으면 브라우즈 판정 → 임베딩 생략, 지역 전체를 고유 블로거 수(다수결) 정렬, `total` 노출(애월 108곳). 해지개 미노출의 근거 있는 답: 상위가 97~98명대, 해지개는 79.
**발견**: 오아시스80이 "휴업(재정비)" 요약 달고 상위 노출 → 폐업·휴업 신호 정렬 강등은 merge.py 몫.

## ④ LLM 층 [2.5+3] — 선별 + 이유

- 임베딩=후보 소집(재현율), LLM=판단(정밀도). gpt-5-mini(json_object, minimal, 4000) — 파이프라인 관례 계승
- 카드 id는 코드 운반(원칙 5) / **인기 수치는 LLM 입력 제외**(원칙 8의 LLM 버전 — LLM도 큰 숫자에 홀린다)
- 브라우즈는 지분 순서 유지+주석만, 조건 질의만 재정렬 허용. 어떤 실패에도 검색 무사(폴백)
- 응답: `intro`(1~2문장) + 카드별 `reason`. 지연 실측 6~9초 → 프론트 로딩 UI 필수, `explain=0` 스위치 있음

## 오늘의 인사이트

1. **층마다 직업이 다르다** — 조회는 사전, 소집은 임베딩, 판단은 LLM, 검증·운반은 코드. 한 층에 다 시키면 해지개 사건이 난다
2. **식별자는 전부 LLM 우회** — video_id(기존)에 이어 지역(주소 유도)·카페 정체성(place_id)까지
3. **빈 질의에는 지분이 순위다** — 인기≠적합성(원칙 8)은 질의에 선호가 있을 때 얘기. 선호가 없으면 다수결이 가장 정직한 순서
4. **층을 옮기면 독이 약이 된다** — 같은 키워드가 임베딩에선 지역 스팸, 라벨층에선 필터·표시 자산
5. **API를 바꿔도 검색엔진은 따라오지 않는다** — 네이버 지도 앱의 검색 품질은 지도 API에 안 담겨 있다. 액자(지도 SDK)와 그림(검색 품질)의 구분
6. **대조는 수단, 번호가 목적** — place_id 발급 과정의 좌표 대조는 신원 확인이었고, 소득은 영구 키 + 중복 검출 + 실존 교차 검증
