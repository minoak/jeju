# W2~W4 지침 — router / relax 사다리 / 종합 (클로드 코드 작업 계약)

> 배경: `백엔드개편_지시서_v2.md`의 W2·W3·W4. 아키텍처 한 문장 —
> **"검색엔진에 LLM을 붙인 게 아니라, LLM에게 검색 도구를 줬다."**
> LLM은 양끝 2회(router/종합), 몸통은 전부 코드. 기존 확정 결정 15·21·22·25·26의 연장선.

## ⓪ 착수 전 확인 (첫 행동 — 건너뛰기 금지)

1. `data/processed/cards.json` 존재 여부 확인 (W0 정본 병합 산출물).
   - **없으면**: W0 미완료. server 통합 시도 금지. 아래 모듈들을 독립 실행 가능하게만 만들고
     mock 카드(`data/mock/cards.json`)로 테스트할 것.
2. 잠금 파일 확인 — 다른 세션이 작업 중일 수 있음: `pipeline/merge.py` `pipeline/embed.py`
   `app/server.py` `FRONTEND_지침.md` `HANDOFF.md` + `pipeline/evidence_polarity.py` `근거극성_지침.md`
   **전부 수정 금지. 이 작업의 산출물은 신규 파일뿐이다.** server.py 연결은 W0 커밋 후 별도 작업.
3. `app/translate.py`(W1 산출물) 존재 확인.
   - **없으면**: 같은 시그니처의 스텁을 `app/translate_stub.py`로 만들어 진행
     (반환 계약: `{"tag", "method": "exact|embedding|unresolved", "score", "input"}`) — W1 완료 시 교체.

## 공통 계약 — TraceState (모든 모듈을 관통하는 상태 객체)

```json
{
  "query": "애월에서 노을 보기 좋은 조용한 카페",
  "intent": {"유형": "조건검색", "지역": "애월", "하드": [], "소프트": ["노을", "조용함"]},
  "translation": [{"input": "노을", "tag": "노을", "method": "exact", "score": 1.0}],
  "unresolved": [],
  "funnel": [{"stage": "전체", "n": 1064}, {"stage": "region:애월", "n": 87},
             {"stage": "tags:노을+조용함", "n": 0}, {"stage": "relax:조용함 제거", "n": 12}],
  "relaxation": [{"action": "soft_drop", "condition": "조용함", "label": "조용함 조건을 빼고 찾았어요"}],
  "region_expanded": null,
  "results": [{"spot_name": "…", "score_parts": {"태그충족": 2, "다수결": 5, "반응": 0.8}}]
}
```
- 각 단계는 자기 결정을 여기 기록만 하고 다음으로 넘긴다. 이 객체가 그대로
  ① LangGraph state(W5) ② /trace 응답 ③ 종합 LLM 입력이 된다. 설명가능성은 후처리가 아니라 데이터다.

## W2 — router (신규 `app/router.py`)

- 역할: 자연어 쿼리 → `intent` (위 스키마). LLM 호출 1회, 창의성 없음 — 구조화 추출만.
- 유형 3분기(확정 결정 21의 LLM 승계): `조회`(특정 카페명) / `브라우즈`(지역만, 조건 없음) / `조건검색`.
- 실패 무해 원칙: LLM 실패·빈 응답 시 기존 코드 3분기 로직으로 폴백 (이름사전 매치 → 브라우즈 → 조건).
- 모델: gpt-5-mini, `max_completion_tokens=4000` + `reasoning_effort="minimal"` (빈 응답 방지 — 실측 함정),
  `response_format={"type":"json_object"}`, RateLimitError 지수 백오프. `.env OPENAI_KEY`.
- 하드/소프트 구분은 태그사전v2의 `hard_capable`을 참조해 **코드가 후처리** (LLM 판정 맡기지 말 것).
  "아이와 함께" 류 감지 시 노키즈존 하드 배제 세팅 (`exclude_for` 의미론).

## W3 — relax 사다리 (신규 `app/relax.py`)

- 발동 조건: 후보 0건일 때만. LLM 호출 0회 — 전부 코드.
- 사다리 (순서 고정, 각 단계마다 재검색 1회):
  1. 소프트조건을 하나 제거 (제거 순서: translation score 낮은 것부터 — 해석 확신이 약한 조건 먼저)
  2. 인접 지역으로 확장 (아래 인접표, 확장 사실을 label로 명시)
  3. 끝. **하드조건은 어떤 경우에도 완화하지 않는다** (주차 필수인 사람에게 주차 없는 곳을 주면 신뢰 끝).
- 모든 완화는 `relaxation[]`에 label과 함께 기록 — 이 label이 그대로 사용자에게 보인다.
- 인접 지역표 초안 (버킷 기준, ⚠ 민옥 지리 검수 전 — 초안 표시 유지):
```json
{"제주시내": ["애월", "조천"], "애월": ["제주시내", "한림"], "한림": ["애월", "대정"],
 "대정": ["한림", "안덕"], "안덕": ["대정", "중문"], "중문": ["안덕", "서귀포시내"],
 "서귀포시내": ["중문", "남원"], "남원": ["서귀포시내", "표선"], "표선": ["남원", "성산"],
 "성산": ["표선", "구좌", "우도"], "구좌": ["성산", "조천"], "조천": ["구좌", "제주시내"],
 "우도": ["성산"]}
```
- 세부 지역(협재→한림, 세화/김녕/월정리/종달/송당→구좌, 함덕→조천, 사계→안덕 등)은
  W0 정본의 버킷/세부 2단 계층을 따른다 — 자체 매핑 만들지 말 것.

## W4 — 종합 (신규 `app/synthesize.py`)

- 입력: TraceState + 카드 5~8장. 출력: 근거 인용 추천문. LLM 호출 1회.
- **세 규칙 (전부 코드로 강제, 프롬프트만 믿지 말 것):**
  1. 카드에 없는 사실 첨가 금지 — 출력의 모든 인용문은 카드 evidence의 **부분문자열인지 코드가 검증**
     (확정 결정 26 패턴). 실패 인용은 해당 문장 제거.
  2. 번역·완화 로그 인용 — relaxation의 label과 unresolved를 반드시 언급
     ("조용함 조건은 빼고 찾았어요" / "'감성적인' 조건은 해석하지 못했어요").
  3. 0건은 0건 — 사다리 끝까지 가도 없으면 "조건에 맞는 카페 없음"을 먼저 말하고,
     완화 label 붙은 대안 제시. 정직한 실패가 컨셉이다.
- 인기 수치(조회수·언급수 원값)는 LLM 입력 금지 (확정 결정 25) — 코드가 정렬에만 사용.
- 근거극성(pro/con) 통합은 **이 작업 스코프 밖** — `근거극성_지침.md` 참조, CP-A 후 별도.

## 검증 (완료 기준 — 신규 `eval/w234_검증.py`)

mock 또는 cards.json으로 시나리오 8종 통과표 출력:
① 조회("카페한라 어때") ② 브라우즈("애월 카페") ③ 조건 정상("애월 오션뷰")
④ 어휘 간극("노을 맛집" → 노을 번역 확인) ⑤ 0건→소프트 완화(label 확인)
⑥ 0건→지역 확장(label 확인) ⑦ 완전 0건(정직 보고) ⑧ 미해석 조건(unresolved 언급)
각 시나리오에서 TraceState 전문을 저장 (`eval/trace_샘플/*.json`) — /trace와 발표 데모 재료.
