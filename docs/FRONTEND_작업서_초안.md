## 작업 1 — 카드 `matched` 조립 표시 (파도 7-2, 지침 180~189행)

`/search` 카드에 오는 결정적 근거를 reason 자리 대체로 한 줄 조립:

```jsonc
"matched": {                 // 근거 없으면 필드 자체가 생략됨 — 없으면 안 그린다
  "region": "애월",           // 질의 지역과 일치했을 때만
  "tags": ["오션뷰", "조용"],  // 질의와 겹친 태그 (최대 4)
  "bloggers": 26              // 고유 블로거 수
}
```

- 표시 형식 제안: `애월 ✓ · 오션뷰·조용 · 블로거 26명 언급` — 세 조각 모두 있을 때만 전부,
  일부만 있으면 있는 것만 (region 없이 tags+bloggers만 등).
- 현재 카드의 `c.reason || firstSentence(c.summary)` 폴백(:746 부근)과의 관계:
  **matched가 있으면 matched 줄이 우선**, 없으면 기존 폴백 유지. reason 필드는 현재 안 옴(파도 7 은퇴)
  — 파도 8에서 부활 예정이므로 폴백 로직은 지우지 말 것.
- 컨셉: "지어낼 수 없는 이유만 표시" — LLM 문장보다 신뢰 신호가 강하다.

**검증 쿼리** (백엔드 :8000 필요): `애월 오션뷰` / `성산 대형 카페` 검색 →
블로거 수 큰 카드(해일리 카페 222명·프릳츠 제주성산점 211명 등)에서 matched 줄 확인.
`curl "http://localhost:8000/search?q=애월 오션뷰"` 로 matched 필드 실재 먼저 확인 가능.

## 작업 2 — `closed` 폐업 배지 (파도 7-2, 지침 190~191행)

```jsonc
"closed": true    // 폐업 — 이름 조회 시에만 등장 (검색·브라우즈엔 안 나옴)
```

- 카드에 "폐업한 곳이에요" 배지 (경고색). 지도 마커에서 제외 권장.
- 컨셉: 지우면 사용자가 모르고 찾아간다 — 안내가 신뢰 (지침 :83 원칙).
- 배지 우선순위 (겹칠 때): `name_match` > `closed` > 여론 뱃지. 폐업 카드는 별점·근거보기보다 배지가 먼저 보여야 한다.

**검증 쿼리**: 폐업 카드 12장 실존 — 이름 조회로만 나온다:
`수리코` / `애월빵공장` / `올댓브라운` / `북카페 휴일` 중 아무거나 검색 → closed 배지 확인.
+ 역검증: `애월 카페` 브라우즈에 애월빵공장이 **안 나오는지** (나오면 백엔드 버그 — 프론트에서 가리지 말고 보고).
